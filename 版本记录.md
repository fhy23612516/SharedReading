# 版本记录

以后每次做“大更新”，都在本文件最上方新增一条版本记录，记录：

1. 更新日期。
2. 本次新增能力。
3. 改动到的关键文件。
4. 部署时需要额外执行的命令。
5. 当前仍未完成或需要注意的事项。

## v0.4.1 - 2026-04-29

本次主题：项目内置 MCP 服务。

新增能力：

1. 新增轻量 stdio MCP 服务 `tools/shared-reading-mcp.js`。
2. MCP 支持读取项目关键文档资源。
3. MCP 支持查询项目状态、输出部署步骤、搜索项目、运行检查。
4. MCP 支持向 `版本记录.md` 顶部追加大更新记录。
5. 新增 `npm run mcp` 启动脚本。
6. 新增 `MCP使用说明.md`，记录客户端配置示例和工具说明。

关键文件：

1. `tools/shared-reading-mcp.js`
2. `MCP使用说明.md`
3. `package.json`
4. `版本记录.md`

部署注意：

1. 这个 MCP 只用于开发辅助，不影响线上服务。
2. 服务器正常部署仍按 `部署文档.md` 执行。
3. 如果要在本地客户端使用，需要手动把 MCP 注册到客户端配置里。

未完成事项：

1. MCP 暂未提供数据库迁移执行能力，只输出部署命令。
2. MCP 暂未接入远程服务器操作。

## v0.4.0 - 2026-04-29

本次主题：导入书籍、阅读器常用功能、正式域名配置文档。

新增能力：

1. 支持登录用户导入 `.txt` 或粘贴正文，导入后可在创建房间时选择。
2. 后端新增 `POST /api/books/import` 和 `GET /api/books/mine`。
3. JSON 和 MySQL 两种存储都支持导入书籍。
4. MySQL 新增 `books` 表，启动时也会自动补建。
5. 阅读页新增字体、字号、行距、主题设置。
6. 阅读页支持选中文字后保存为划线。
7. 阅读页支持按当前进度添加书签/标签，并点击书签跳转。
8. Nginx 配置模板写入正式域名 `shareread.heiheihei.pw`。
9. API 冒烟测试新增导入书籍和导入内容创建房间覆盖。

关键文件：

1. `server.js`
2. `app.js`
3. `styles.css`
4. `schema/mysql.sql`
5. `deploy/nginx.shared-reading.conf`
6. `tests/api-smoke.test.js`
7. `接口文档.md`
8. `部署文档.md`
9. `服务器配置记录与小程序迁移.md`

部署注意：

1. 服务器需要执行 `git pull`。
2. 因为之前已经加入 `mysql2`，服务器仍建议执行 `npm install`。
3. 如果已启用 MySQL，重启后后端会自动创建 `books` 表；也可以重新执行 `sudo mysql < schema/mysql.sql`。
4. Nginx 需要重新复制模板或手动把 `server_name` 改成 `shareread.heiheihei.pw`。
5. 修改 Nginx 后执行 `sudo nginx -t && sudo systemctl reload nginx`。

未完成事项：

1. 导入书籍目前是纯文本导入，不支持 EPUB/PDF 解析。
2. 划线和书签当前保存在浏览器本地，暂未同步到账号或小程序。
3. 反馈后台管理仍未实现。

## v0.2.0 - 2026-04-28

本次主题：前后端分离运行、服务器部署、ngrok 和弱网体验。

新增能力：

1. 前端服务 `frontend.server.js` 和后端服务 `server.js` 分开运行。
2. 本地一键启动/停止脚本。
3. 后端开发热更新脚本 `start-dev`。

5. 弱网下进度和消息先本地显示，再通过 outbox 自动补发。
6. SSE 局部事件推送，减少全量房间刷新。
7. 服务器部署文档、PM2 配置和 Nginx 模板。

关键文件：

1. `frontend.server.js`
2. `start-all.ps1`
3. `stop-all.ps1`
4. `start-dev.ps1`
5. `部署文档.md`
6. `服务器配置记录与小程序迁移.md`
