本文档记录本次从本地项目到阿里云服务器的完整配置过程，并说明当前后端如何作为小程序后端使用。

当前 `server.js` 可以作为小程序后端的基础版本使用。

```text
https://shareread.heiheihei.pw
```

已经具备：

1. 房间创建接口。
2. 房间加入接口。
3. 房间详情接口。
4. 阅读进度上报接口。
5. 聊天消息发送接口。

5. 如果启用 MySQL，需要完成数据库初始化、备份策略和旧 JSON 数据迁移。

7. 如果使用中国大陆服务器和正式域名，按平台要求处理域名备案、证书和小程序审核。
8. 小程序端需要补书架、浏览记录、搜索、评论的原生页面。

## 2. 当前服务器结构

当前网页端部署结构：

```text
浏览器 / 手机
  |
  | HTTP 80
  v
阿里云安全组
  |
  v
Nginx
  |
  |-- /api/ -> server.js:3210
  |              |
  |              v
  |        store.json 或 MySQL
  |
  |-- / -> frontend.server.js:3211
```

端口职责：

1. `80`：公网 HTTP 入口，由 Nginx 监听。
2. `443`：公网 HTTPS 入口，配置证书后由 Nginx 监听。
3. `3211`：前端服务，只监听服务器本机 `127.0.0.1`。
4. `3210`：后端 API 服务，只监听服务器本机 `127.0.0.1`。

不要把 `3210`、`3211` 直接开放到公网。

## 7. Nginx 配置过程

复制配置：

```bash
sudo cp deploy/nginx.shared-reading.conf /etc/nginx/sites-available/shared-reading
sudo ln -s /etc/nginx/sites-available/shared-reading /etc/nginx/sites-enabled/shared-reading
```

如果还没有域名，先把配置中的：

```nginx
server_name your-domain.com;
```

改成：

```nginx
server_name _;
```

如果已经有域名，则改成真实域名：

```nginx
server_name shareread.heiheihei.pw;
```

检查并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

本次遇到的问题：

```text
conflicting server name "_" on 0.0.0.0:80, ignored
```

原因：Nginx 默认站点和当前站点都使用了 `server_name _;`。

解决：

```bash
sudo rm -f /etc/nginx/sites-enabled/default
sudo rm -f /etc/nginx/sites-enabled/shared-reading
sudo ln -s /etc/nginx/sites-available/shared-reading /etc/nginx/sites-enabled/shared-reading
sudo nginx -t
sudo systemctl reload nginx
```

验证 Nginx 是否代理到项目：

```bash
curl http://127.0.0.1 | head -20
```

正确时应该看到：

```html
<title>一起等你读</title>
```

## 8. 阿里云安全组配置

阿里云控制台路径：

```text
ECS -> 实例 -> 选择服务器 -> 安全组 -> 访问规则 -> 入方向
```

至少放行：

```text
协议：TCP
端口：80/80
授权对象：0.0.0.0/0
描述：HTTP
```

以后配置 HTTPS 后再放行：

```text
协议：TCP
端口：443/443
授权对象：0.0.0.0/0
描述：HTTPS
```

SSH 端口不要删除：

```text
协议：TCP
端口：22/22
授权对象：你的公网 IP/32 或 0.0.0.0/0
```

不要开放：

```text
3210
3211
```

## 9. HTTP 与 HTTPS

当前 HTTP 可访问：

```text
http://服务器公网 IP
```

HTTPS 暂时不可用是正常的，因为还没有域名证书。

要使用 HTTPS，需要：

1. 准备域名。
2. 域名 A 记录解析到服务器公网 IP。
3. 阿里云安全组放行 `443`。
4. 在服务器执行 Certbot。

命令：

```bash
sudo certbot --nginx -d shareread.heiheihei.pw
```

成功后访问：

```text
https://shareread.heiheihei.pw
```

不建议用裸 IP 配 HTTPS，免费证书通常绑定域名。

## 10. 小程序后端迁移方案

小程序端推荐直接调用同一域名下的 `/api`：

```text
https://你的域名/api/bootstrap
https://你的域名/api/rooms
https://你的域名/api/rooms/:roomId/progress
https://你的域名/api/rooms/:roomId/messages
https://你的域名/api/search?q=关键词
https://你的域名/api/bookshelf
https://你的域名/api/reading/history
https://你的域名/api/stories/:storyId/comments
```

当前 Nginx 推荐路径是：

```text
/api -> 127.0.0.1:3210
/ -> 127.0.0.1:3211
```

这样小程序调用接口时会直接到后端，减少一层前端服务转发。对应模板已经写在 `deploy/nginx.shared-reading.conf`：

```nginx
location /api/ {
  proxy_pass http://127.0.0.1:3210;
  proxy_http_version 1.1;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header X-Forwarded-Proto $scheme;
  proxy_buffering off;
  proxy_cache off;
  proxy_read_timeout 3600s;
  proxy_send_timeout 3600s;
}

location / {
  proxy_pass http://127.0.0.1:3211;
  proxy_http_version 1.1;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header X-Forwarded-Proto $scheme;
}
```

小程序端注意：

1. 使用 `wx.request` 请求普通 API。
2. 登录 token、`roomId`、离线队列要用小程序本地存储保存。
3. 小程序后台需要配置合法请求域名。
4. 如果小程序环境不适合直接使用 SSE，需要把实时能力改成 WebSocket 或轮询。
5. 当前后端已有账号登录态；房间邀请码和房间访问 token 可以后续继续加强。
6. 如需 MySQL 存储，按 `MySQL迁移与启用.md` 配置。
7. 书架、浏览记录和发布评论需要登录 token。

## 12. 常用排查命令

检查 PM2：

```bash
pm2 status
pm2 logs shared-reading-api
pm2 logs shared-reading-web
```

检查本机服务：

```bash
curl -v http://127.0.0.1:3211
curl -v http://127.0.0.1:3210/api/bootstrap
curl -v http://127.0.0.1
```

检查 Nginx：

```bash
sudo nginx -t
sudo systemctl status nginx --no-pager
sudo ss -lntp | grep -E ':80|:443|:3211|:3210'
```

判断问题位置：

1. `127.0.0.1:3211` 不通：前端 PM2 服务异常。
2. `127.0.0.1:3210/api/bootstrap` 不通：后端 PM2 服务异常。
3. `127.0.0.1` 返回 Nginx 默认页：Nginx 站点配置冲突。
4. 服务器本机正常但公网打不开：阿里云安全组或系统防火墙问题。
5. HTTP 正常但 HTTPS 不正常：还没有配置域名证书或 `443` 未放行。
