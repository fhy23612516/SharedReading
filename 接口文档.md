# 一起阅读 API 接口约束

本文档面向后续小程序迁移。当前网页端通过前端服务代理访问 `/api`，小程序端可以直接请求同一套后端接口。

## 基础约定

1. 默认后端地址：`http://127.0.0.1:3210`
2. 公网访问时：由前端服务或网关代理到后端 `/api`
3. 请求格式：`Content-Type: application/json`
4. 身份字段：推荐使用 `Authorization: Bearer <token>`；旧网页端仍兼容 `userId` 和 `name`
5. 房间身份：后端会校验 token 用户或 `userId` 是否属于房间成员
6. 消息幂等：发送消息建议带 `clientId`，用于弱网重试去重

## 通用错误

| 错误码 | 含义 |
| --- | --- |
| `not_found` | 接口不存在 |
| `room_not_found` | 房间不存在 |
| `not_room_member` | 当前身份不是房间成员 |
| `not_room_owner` | 当前身份不是房主 |
| `room_full` | 房间已满 |
| `room_ended` | 房间已结束或已关闭 |
| `invalid_progress` | 进度值非法 |
| `empty_message` | 消息为空 |
| `message_too_long` | 消息过长 |
| `unauthorized` | 未登录或 token 无效 |
| `invalid_credentials` | 账号或密码错误 |
| `account_exists` | 账号已存在 |
| `invalid_account` | 账号格式非法 |
| `weak_password` | 密码长度不足 |
| `invalid_feedback_type` | 反馈类型非法 |
| `feedback_content_required` | 反馈内容为空 |
| `feedback_too_long` | 反馈内容过长 |
| `book_title_required` | 导入书籍缺少书名 |
| `book_content_too_short` | 导入正文太短 |
| `book_content_too_long` | 导入正文太长 |

## 接口列表

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `GET` | `/api/bootstrap` | 获取内容列表、进度参考线、快捷消息 |

| `POST` | `/api/session` | 创建或恢复用户身份 |

| `POST` | `/api/auth/login` | 登录自有账号 |

## 关键请求体

### 创建或恢复身份

```json
{
  "userId": "可选，已有身份时传",
  "name": "昵称"
}
```

### 创建房间

```json
{
  "userId": "user-xxx",
  "name": "昵称",
  "storyId": "story-xxx",
  "threshold": 8
}
```

### 加入房间

```json
{
  "userId": "user-xxx",
  "name": "昵称",
  "code": "123456"
}
```

### 上报进度

```json
{
  "userId": "user-xxx",
  "name": "昵称",
  "progress": 42.5
}
```

### 发送消息

```json
{
  "userId": "user-xxx",
  "name": "昵称",
  "clientId": "local-message-id",
  "content": "消息内容"
}
```

如果已经登录，创建房间、加入房间、上报进度、发送消息等写接口可以不再传 `userId` 和 `name`，改用统一请求头：

```http
Authorization: Bearer session-token
```

## SSE 事件

| 事件名 | 含义 |
| --- | --- |
| `room` | 房间全量状态，进入房间或房间关闭/完成时使用 |
| `progress` | 某个用户的进度变化 |
| `message` | 新聊天消息 |
| `presence` | 成员上线、离线、返回 |
| `ping` | 保活事件 |

## 小程序迁移注意事项

1. 小程序不能直接依赖浏览器 `localStorage`，需要使用小程序本地存储保存 `userId`、`roomId` 和离线队列。
2. 如果小程序环境不支持标准 `EventSource`，需要改用 WebSocket 或轮询。
3. 公网部署时必须使用 HTTPS 域名，并在小程序后台配置合法请求域名。
4. 当前后端已经支持自有账号 token；小程序端可以用同一套账号接口登录，不强制微信/QQ 登录。

## 当前后端作为小程序后端的判断

当前 `server.js` 可以作为小程序后端的基础版本使用，普通 API 可以被小程序通过 `wx.request` 调用。
账号、反馈、房间和进度接口已经统一在 `/api` 下，生产环境推荐让 Nginx 把 `/api/` 直接代理到 `127.0.0.1:3210`。

建议的小程序调用域名形式：

```text
https://你的域名/api/...
```

正式迁移前建议补充：

1. HTTPS 域名和小程序后台合法域名配置。
2. 小程序端 token 本地存储和失效处理。
3. 如果需要实时体验，优先评估 WebSocket；如果先做 MVP，可以用 1-2 秒轮询房间详情。
4. 服务器按需启用 MySQL，步骤见 `MySQL迁移与启用.md`。

完整服务器配置和迁移记录见 `服务器配置记录与小程序迁移.md`。

### 导入书籍

导入书籍需要登录账号。

```text
POST /api/books/import
```

```http
Authorization: Bearer session-token
```

```json
{
  "title": "我的故事",
  "author": "可选作者",
  "tags": "小说 睡前读物",
  "summary": "可选简介",
  "text": "第一段正文。\n\n第二段正文。"
}
```

返回：

```json
{
  "book": {
    "id": "book_xxx",
    "title": "我的故事",
    "author": "可选作者",
    "cover": "导入书籍",
    "summary": "可选简介",
    "body": ["第一段正文。", "第二段正文。"],
    "wordCount": 12,
    "source": "imported",
    "tags": ["小说", "睡前读物"]
  }
}
```

导入后的书会出现在当前账号的 `/api/bootstrap` 内容列表里，创建房间时直接把 `book.id` 作为 `storyId` 传给 `/api/rooms`。
