# 一起阅读 API 接口约束

本文档面向后续小程序迁移。当前网页端通过前端服务代理访问 `/api`，小程序端可以直接请求同一套后端接口。

## 基础约定

1. 默认后端地址：`http://127.0.0.1:3210`
2. 公网访问时：由前端服务或网关代理到后端 `/api`
3. 请求格式：`Content-Type: application/json`
4. 身份字段：需要写操作时传 `userId` 和 `name`
5. 房间身份：后端会校验 `userId` 是否属于房间成员
6. 消息幂等：发送消息建议带 `clientId`，用于弱网重试去重

## 通用错误

| 错误码 | 含义 |
| --- | --- |
| `not_found` | 接口不存在 |
| `room_not_found` | 房间不存在 |
| `not_room_member` | 当前身份不是房间成员 |
| `not_room_owner` | 当前身份不是房主 |
| `room_full` | 房间已满 |
| `room_ended` | 房间已结束或已关闭 |
| `invalid_progress` | 进度值非法 |
| `empty_message` | 消息为空 |
| `message_too_long` | 消息过长 |

## 接口列表

| 方法 | 路径 | 用途 |
| --- | --- | --- |
| `GET` | `/api/bootstrap` | 获取内容列表、进度参考线、快捷消息 |
| `GET` | `/api/records` | 获取最近共读记录 |
| `POST` | `/api/session` | 创建或恢复用户身份 |
| `POST` | `/api/rooms` | 创建房间 |
| `POST` | `/api/rooms/join` | 通过房间码加入房间 |
| `GET` | `/api/rooms/:roomId` | 获取房间详情 |
| `GET` | `/api/rooms/:roomId/events` | 建立 SSE 实时连接 |
| `POST` | `/api/rooms/:roomId/presence` | 上报在线心跳 |
| `POST` | `/api/rooms/:roomId/progress` | 上报阅读进度 |
| `POST` | `/api/rooms/:roomId/messages` | 发送聊天消息 |
| `POST` | `/api/rooms/:roomId/complete` | 标记自己读完 |
| `POST` | `/api/rooms/:roomId/leave` | 离线/返回首页 |
| `POST` | `/api/rooms/:roomId/close` | 房主关闭房间 |

## 关键请求体

### 创建或恢复身份

```json
{
  "userId": "可选，已有身份时传",
  "name": "昵称"
}
```

### 创建房间

```json
{
  "userId": "user-xxx",
  "name": "昵称",
  "storyId": "story-xxx",
  "threshold": 8
}
```

### 加入房间

```json
{
  "userId": "user-xxx",
  "name": "昵称",
  "code": "123456"
}
```

### 上报进度

```json
{
  "userId": "user-xxx",
  "name": "昵称",
  "progress": 42.5
}
```

### 发送消息

```json
{
  "userId": "user-xxx",
  "name": "昵称",
  "clientId": "local-message-id",
  "content": "消息内容"
}
```

## SSE 事件

| 事件名 | 含义 |
| --- | --- |
| `room` | 房间全量状态，进入房间或房间关闭/完成时使用 |
| `progress` | 某个用户的进度变化 |
| `message` | 新聊天消息 |
| `presence` | 成员上线、离线、返回 |
| `ping` | 保活事件 |

## 小程序迁移注意事项

1. 小程序不能直接依赖浏览器 `localStorage`，需要使用小程序本地存储保存 `userId`、`roomId` 和离线队列。
2. 如果小程序环境不支持标准 `EventSource`，需要改用 WebSocket 或轮询。
3. 公网部署时必须使用 HTTPS 域名，并在小程序后台配置合法请求域名。
4. 当前 MVP 身份是轻量身份，不是正式登录系统；如果要长期公网使用，需要增加房间访问 token 或账号登录。
