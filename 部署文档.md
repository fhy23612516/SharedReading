# 一起阅读服务器部署文档

推荐部署方式：GitHub 拉代码 + PM2 守护 Node 服务 + Nginx 反向代理 + HTTPS。

## 1. 推到 GitHub 前

本项目已经配置 `.gitignore`，不会提交以下本地运行数据：

1. `.runtime/`
2. `data/store.json`
3. `.env`
4. `node_modules/`

你需要提交代码文件、文档、`data/stories.js`、`ecosystem.config.cjs` 和 `deploy/nginx.shared-reading.conf`。

## 2. 服务器准备

以下命令以 Ubuntu 为例：

```bash
sudo apt update
sudo apt install -y git nginx curl
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt install -y nodejs
sudo npm install -g pm2
```

创建运行数据目录：

```bash
sudo mkdir -p /var/lib/shared-reading
sudo chown -R $USER:$USER /var/lib/shared-reading
```

## 3. 拉取 GitHub 代码

```bash
cd /opt
sudo git clone https://github.com/你的用户名/你的仓库名.git shared-reading
sudo chown -R $USER:$USER /opt/shared-reading
cd /opt/shared-reading
```

如果你的仓库是私有仓库，服务器需要先配置 GitHub SSH key，或者使用 GitHub token 拉取。

## 4. 启动服务

当前项目包含 MySQL 可选存储依赖 `mysql2`，服务器首次部署和 `package.json` 变化后都需要安装依赖：

```bash
npm install
npm test
```

启动 PM2：

```bash
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
```

查看状态：

```bash
pm2 status
pm2 logs shared-reading-api
pm2 logs shared-reading-web
```

## 5. 配置 Nginx

复制模板：

```bash
sudo cp deploy/nginx.shared-reading.conf /etc/nginx/sites-available/shared-reading
sudo ln -s /etc/nginx/sites-available/shared-reading /etc/nginx/sites-enabled/shared-reading
```

编辑域名：

```bash
sudo nano /etc/nginx/sites-available/shared-reading
```

把：

```nginx
server_name your-domain.com;
```

改成你的真实域名，例如：

```nginx
server_name read.example.com;
```

检查并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 6. 配 HTTPS

推荐使用 Certbot：

```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d read.example.com
```

证书配置完成后，公网访问：

```text
https://read.example.com
```

## 7. 更新代码

以后你在本地推送 GitHub 后，服务器执行：

```bash
cd /opt/shared-reading
git pull
npm install
pm2 restart ecosystem.config.cjs
```

如果只改了前端静态文件，也可以直接重启 `shared-reading-web`：

```bash
pm2 restart shared-reading-web
```

如果改了后端：

```bash
git pull
npm install
pm2 restart shared-reading-api
```

如果确定本次没有改 `package.json`，可以跳过 `npm install`。

## 8. 端口说明

公网只开放：

1. `80`
2. `443`

不要对公网开放：

1. `3210`
2. `3211`

这两个端口只在服务器本机监听 `127.0.0.1`，由 Nginx 转发访问。

当前 Nginx 模板已经把 `/api/` 直接代理到后端 `3210`，页面请求代理到前端 `3211`。这样小程序调用 `/api` 时不会再多经过一层前端服务。

## 9. MySQL 可选存储

默认部署仍使用 JSON 文件：

```text
STORAGE_DRIVER=json
STORE_PATH=/var/lib/shared-reading/store.json
```

如果要启用 MySQL，按 [MySQL迁移与启用.md](</E:/Program Files/VibeCoding/SharedReading/MySQL迁移与启用.md>) 执行，核心步骤是安装 MySQL、初始化 `schema/mysql.sql`、把后端环境变量改为 `STORAGE_DRIVER=mysql`，然后重启 PM2。

## 10. 常见问题

### 页面能打开，但消息和进度不同步

检查 Nginx 是否保留了 SSE 配置：

```nginx
proxy_buffering off;
proxy_read_timeout 3600s;
```

### PM2 显示服务在重启

查看日志：

```bash
pm2 logs shared-reading-api
pm2 logs shared-reading-web
```

### 最近记录丢失

确认 `STORE_PATH` 指向的是持久化路径：

```text
/var/lib/shared-reading/store.json
```

不要把运行数据放在会被重新部署覆盖的目录里。
