# 导入功能改造方向

本文记录当前导入书籍功能的已知问题和后续改造方向。

## 当前实现状态（v0.8.0）

已经完成：

1. Web 导入页支持 `UTF-8` 和 `GBK / GB18030` 编码选择。
2. 选择 TXT 后按所选编码读取，正文自动填入，书名为空时自动使用文件名。
3. 导入前显示字数、段落数、章节识别数量、请求体大小估算和风险提示。
4. 前端提交前校验最少 30 字、正文 50 万字符上限和约 2.5MB 请求体上限。
5. 后端导入接口单独放宽请求体上限到约 2.5MB，避免中文长文本未进入业务校验就失败。
6. 导入按钮增加提交中状态，并补充请求体过大、JSON 异常、请求超时等错误文案。
7. 超过单次导入上限的大书会自动走分章导入。
8. 分章导入新增 `book_chapters` 存储，章节可作为虚拟 `storyId` 创建共读房间。
9. 创建房间时选择分章书籍会加载目录并选择具体章节。
10. 前端识别不到章节标题时，会按固定长度自动切片。

仍未完成：

1. 暂不支持 EPUB/PDF。
2. 暂不支持完整书籍详情页、目录页和跨章节连续阅读。
3. 暂不支持导入历史、重新编辑、删除、去重提醒和内容治理。

## 当前问题

1. 读取进度和取消导入还没有做：长文本本地读取时只有最终结果，没有百分比进度。
2. 内容治理缺失：当前只做长度限制，没有敏感词、重复内容、异常文本或垃圾内容检查。
3. 导入后的后续动作还不完整：当前导入成功后跳转创建房间，还没有“加入书架”“继续导入”“查看书籍详情”的独立成功页。
4. 分章书籍还没有正式详情页和目录页，当前只在创建房间时加载目录。
5. 跨章节连续阅读、整本书进度和章节间跳转还没有做。

## 优先改造顺序

### P0：让导入稳定可用

1. 前端读取文件时优先尝试 `utf-8`，乱码明显时提示用户切换编码。
2. 增加编码选项：`UTF-8`、`GBK/GB18030`。
3. 前端在提交前检查原始字符数和估算请求体大小。
4. 后端把导入接口请求体限制提高到与业务上限一致，或把业务上限降到真实可提交范围。
5. 导入按钮增加 loading 状态，提交中禁止重复点击。
6. 明确错误文案：未登录、正文太短、正文太长、请求体过大、网络失败、JSON 无效。

### P1：让导入结果可控

1. 增加导入预览页或预览区域。
2. 展示识别出的标题、作者、字数、段落数、章节数。
3. 支持用户手动调整标题、作者、标签和简介。
4. 增加章节识别规则：
   - `第十二章 标题`
   - `第十二回 标题`
   - `Chapter 12`
   - 空行分段兜底
5. 导入后进入“导入成功”页，提供“创建房间”“加入书架”“继续导入”。

### P2：让导入适合长期使用

1. 支持 EPUB 解析。
2. 支持按章节存储，避免整本书作为一个超长正文。
3. 支持书籍详情页和目录页。
4. 支持导入历史、重新编辑、删除导入书籍。
5. 支持同名书籍去重提醒。
6. 支持导入后的封面生成或用户自定义封面。

## 推荐技术方案

1. 前端读取文件为 `ArrayBuffer`，再用 `TextDecoder` 按用户选择编码解码。
2. 前端先本地完成解析和预览，再提交结构化结果。
3. 后端导入接口拆成两层：
   - `POST /api/books/import/preview`：可选，返回解析结果。
   - `POST /api/books/import`：保存确认后的书籍。
4. MySQL 长期方案应从 `books.body_json` 进一步拆出章节表，例如 `book_chapters`。
5. 如果后续做小程序端导入，优先从云端文件或粘贴文本开始，不先做复杂本地文件解析。

## 下一步建议

P0 已完成，P1/P2 中的“按章节存储”和“创建房间选章节”已经完成基础版。下一步建议做书籍详情页、目录页、章节间跳转和整本书阅读进度，再考虑 EPUB。内容治理和导入历史适合在 MySQL 正式稳定后继续补。
